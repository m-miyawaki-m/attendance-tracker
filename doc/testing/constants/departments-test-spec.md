# `constants/departments.ts` 単体テスト仕様書

## 1. テストの目的

`constants/departments.ts`で定義されている部署関連の定数が、アプリケーション全体で一貫して使用される値を正しく提供していることを保証する。

## 2. 使用するライブラリ

* **テストランナー**: Vitest

## 3. テストケース

### 3.1. DEPARTMENTS 定数

| テストNo. | テストケース | 期待する値 |
| :--- | :--- | :--- |
| 1-1 | 配列の長さ | 7要素 |
| 1-2 | 営業部 | `DEPARTMENTS[0]` === `'営業部'` |
| 1-3 | 開発部 | `DEPARTMENTS[1]` === `'開発部'` |
| 1-4 | 総務部 | `DEPARTMENTS[2]` === `'総務部'` |
| 1-5 | 管理部 | `DEPARTMENTS[3]` === `'管理部'` |
| 1-6 | 人事部 | `DEPARTMENTS[4]` === `'人事部'` |
| 1-7 | 経理部 | `DEPARTMENTS[5]` === `'経理部'` |
| 1-8 | マーケティング部 | `DEPARTMENTS[6]` === `'マーケティング部'` |

### 3.2. 型定義

| テストNo. | テストケース | 期待する結果 |
| :--- | :--- | :--- |
| 2-1 | as const | `DEPARTMENTS`がreadonlyタプル型として定義されている。 |
| 2-2 | Department型 | `typeof DEPARTMENTS[number]`が正しいユニオン型を生成する。 |

### 3.3. 使用例検証

| テストNo. | テストケース | 入力 | 期待する結果 |
| :--- | :--- | :--- | :--- |
| 3-1 | 部署存在チェック | `'営業部'` | `DEPARTMENTS.includes('営業部')` === `true` |
| 3-2 | 部署不在チェック | `'存在しない部署'` | `DEPARTMENTS.includes('存在しない部署')` === `false` |
