# `EmployeeListView.vue` 単体テスト仕様書

## 1. 概要

| 項目 | 内容 |
|:---|:---|
| 対象ファイル | `src/views/admin/EmployeeListView.vue` |
| 関連設計書 | [基本設計書 1.4.2](../../design/03-basic/03-basic-design.md#142-従業員管理画面-employeelistviewvue) |
| 設計書記載 | 従業員管理画面（対象月選択、従業員一覧表示、月次勤怠ステータス集計表示） |

## 2. テストの目的

`EmployeeListView.vue`コンポーネントが、従業員一覧と月次勤怠集計を正しく表示し、月選択機能が正しく動作することを保証する。

## 3. 使用するライブラリ

* **テストランナー**: Vitest
* **コンポーネントテスト**: Vue Test Utils
* **Firestore**: モック化されたFirestoreモジュール

## 4. テストケース

### 4.1. 初期表示

| テストNo. | テストケース | 期待する結果 |
| :--- | :--- | :--- |
| 1-1 | 月選択フィールド | 現在の年月が初期値として設定される（`YYYY-MM`形式）。 |
| 1-2 | ヘッダー表示 | 「従業員管理」タイトルが表示される。 |
| 1-3 | テーブルヘッダー | 社員番号、名前、役職、当月の出勤ステータス、備考のカラムが表示される。 |

### 4.2. データ取得

| テストNo. | テストケース | 期待する結果 |
| :--- | :--- | :--- |
| 2-1 | ユーザー取得 | マウント時に`users`コレクションからデータが取得される。 |
| 2-2 | 勤怠取得 | 選択月の開始日〜終了日で勤怠データが取得される。 |
| 2-3 | ローディング状態 | データ取得中は`loading`がtrue、完了後falseになる。 |

### 4.3. 月選択変更 (`watch selectedMonth`)

| テストNo. | テストケース | 操作 | 期待する結果 |
| :--- | :--- | :--- | :--- |
| 3-1 | 月変更時の再取得 | 月選択を変更 | `fetchAttendances`が呼び出され、新しい月のデータが取得される。 |
| 3-2 | 日付範囲計算 | 2026-01を選択 | `startDate: '2026-01-01'`、`endDate: '2026-01-31'`でクエリされる。 |

### 4.4. 従業員リスト計算 (`employeeList` computed)

| テストNo. | テストケース | 入力データ | 期待する結果 |
| :--- | :--- | :--- | :--- |
| 4-1 | 従業員フィルタリング | `role: 'employee'`のみ | `role: 'admin'`のユーザーは除外される。 |
| 4-2 | 勤怠ステータス集計 | ユーザーの月間勤怠 | `attendanceStatus`オブジェクトに各ステータスの件数が設定される。 |
| 4-3 | ステータス初期値 | 勤怠データなし | `{ present: 0, late: 0, early_leave: 0, absent: 0 }`が設定される。 |
| 4-4 | 社員番号表示 | `employeeNumber`がundefined | `'-'`が表示される。 |
